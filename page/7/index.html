Using an array of materials, and setting material index values in three.js

When working with a mesh in three.js a single instance of some kind of mesh material can be passed to the mesh constructor as the second argument. This is fine if you are okay with every face in the geometry being skinned with the same material, otherwise you might want to pass an array of materials instead. When working with an array of materials there is a property of a face3 instance in the geometry of the mesh that is of interest when setting the material index property of the faces. In this post I will be covering some basic demos of how to work with more than one material, and how to go about setting the material index values of a geometry.

The Points material in three.js

The use of Vector3 class instances in three.js is a major part of the process of doing much of anything in three.js. Everything that will be placed in a Scene will contain at least a few points in space made with Vector3, typically combined with a collection of Face3 instances to help compose a geometry representing some kind of solid geometric object. However what if you just want to work with a collection of vectors, and have some kind of way of just displaying some points in space? For this there is the Points constructor that can be used with the Special Points Material that is put in place just for this purpose.
